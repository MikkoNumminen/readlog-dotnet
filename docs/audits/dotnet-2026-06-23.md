## Findings by area

### A — DI & composition
None. The only `AddSingleton` is the stateless `BookDescriptionSanitizer` (no captured scoped dependency — A1 clear); external HTTP uses typed `AddHttpClient<…>` clients (the factory pattern — A2 clear); `DbContext` is `AddDbContext` (scoped), services are `AddScoped`, cache/sanitizer are singletons (A3 clear); the two `Configuration["Authentication:Google:*"]` reads are startup composition deciding whether to register Google login, not business-code string-key access (A4 clear).

### B — EF Core & data access
None. **B1:** all five flagged read queries are **projections** (`.Select(… => new …Dto)` before `ToListAsync`) — EF Core does not change-track projection results, so `AsNoTracking()` would be noise, not a fix. **B2:** the `.ToList()`/`.ToArray()` sites are in-memory maps over already-materialised bounded API responses, not `IQueryable` materialised-then-filtered. **B3:** every async EF call forwards the request `CancellationToken`; the sole `CancellationToken.None` (`GetRecentPublicReadsAsync`) is the documented cache-populate that outlives the request (CLAUDE.md invariant). **B4:** the one `Task.WhenAll` (`BookSearchService`) awaits two **HTTP** clients, not two operations on one `DbContext`. **B5:** the public feed has `.Take(PublicFeedSize = 20)`; library/search queries are owner-bounded (`UserId == userId`).

### C — async/await & concurrency
**Not dispatched — 0 candidates.** No `.Result` / `.Wait()` / `.GetAwaiter().GetResult()`, no `async void`, no unobserved fire-and-forget anywhere in `src`.

### D — web surface
None. **D1:** `@Html.Raw(Model.SafeDescriptionHtml)` renders `_sanitizer.Sanitize(description)` output (Ganss.Xss `HtmlSanitizer`, per the documented XSS invariant). **D2:** all four `[BindProperty]` bind input DTOs/`InputModel`s (`LogBookRequest`, `UpdateReadEntryRequest`, …), never EF entities — no overposting surface. **D3:** no GET handler mutates state; `Logout.OnGet` only `RedirectToPage` (sign-out is POST-only — the documented CSRF invariant). **D4:** every post-auth redirect is `LocalRedirect` (open-redirect-safe).

### E — error handling & resource lifecycle
None. The two `catch (DbUpdateException)` blocks are the documented unique-index/race handlers (re-throw when there's no winner); `BookSearchService` catches `OperationCanceledException` and **re-throws** before its broad `catch (Exception)` logs-and-degrades to `[]` (cancellation is never swallowed — E2 clear); `Log.OnPost` catches the specific `DuplicateReadEntryException` for a friendly model error. No owned `IDisposable` is left undisposed (E3: 0 candidates).
